## 一、当前架构 vs 优化架构

### 当前侧边栏

```
AI 工作台
  🎬 智能创作        ← AI autopilot
  ✂️ 批量混剪        ← 素材 + AI 混剪
  🎞️ 成品库          ← 下载成片
  🎙️ 音频生成        ← TTS 配音

运营管理
  📅 发布排期        ← 排期管理
  🚀 发布中心        ← 文案复制 + 发布
  👤 账户库          ← 账号管理

⚙️ API 设置
```

### 优化后（建议）

```
📊 工作台首页        ← Dashboard：今日数据 / 创作记录一览

内容生产
  🎬 智能创作        ← AI 全自动生成（对标→脚本→视频→文案）
  📂 素材库          ← 本地/云端素材管理（新增独立页）
  ✂️ 批量混剪        ← 素材 + AI 视频混剪+合成

内容管理
  🎞️ 成品库          ← 已合成成片，预览/下载
  🚀 发布中心        ← 视频+文案配对，一键复制发手机
  📅 发布排期        ← 批量排期队列

账号运营
  👤 账号库          ← TikTok 账号池管理
  📈 数据分析        ← 播放量/转化率看板（远期）

⚙️ 系统设置
```

## 二、用户操作流程（商业化标准）

```
第1步：打开 🎬 智能创作
  → 填写产品名、卖点、目标用户
  → 上传对标视频（可选）
  → 点击「AI 全自动生成」
  → 等待 9 步流水线完成
  → 系统自动保存：脚本 + 视频任务 + 发布文案

第2步：进 ✂️ 批量混剪
  → AI 视频生成完毕后
  → 点击「合成完整成片」
  → ffmpeg 自动混剪 → 存入成品库

第3步：进 🚀 发布中心
  → 顶部下拉：选择刚才的创作记录
  → 自动加载：标题 + 正文 + Hashtags
  → 逐个点击「复制」按钮
  → 在手机上打开 TikTok → 粘贴

第4步：手机端发布
  → 电脑端下载视频 → AirDrop 到手机
  → 打开 TikTok → 上传视频 → 粘贴文案 → 发布
```

### 关键设计原则

| 原则 | 说明 |
|------|------|
| **电脑产、手机发** | 中国电脑生产内容 → 美国手机发布，不需要复杂传输 |
| **所见即所拷** | 标题、正文、标签分块，各自独立复制按钮 |
| **一键全拷** | 同时提供「一键全部复制」按钮 |
| **创作-文案联动** | AI 生成视频的同时生成文案，自动关联 |
| **不依赖第三方** | 不需要微信/QQ/WhatsApp 传文案，直接网页复制 |

## 三、已实现的 API 功能

| API 端点 | 用途 | 模型 |
|----------|------|------|
| `/api/ai/autopilot` (SSE) | AI 全自动流水线 | 豆包 LLM + Seedance |
| `/api/ai/generate-caption` | 爆款文案生成 | 豆包 LLM |
| `/api/ai/strategy` | 对标账号分析 | 豆包 LLM |
| `/api/ai/production-script` | 分镜脚本生成 | 豆包 LLM |
| `/api/video/create` | 提交 Seedance 视频生成 | 豆包 Seedance |
| `/api/video/poll` | 轮询 Seedance 任务状态 | Seedance API |
| `/api/video/download` | 下载生成好的视频 | Seedance API |
| `/api/tts/create` | TTS 配音 | 火山 TTS |
| `/api/mixcut/render` | ffmpeg 混剪合成 | ffmpeg |
| `/api/mixcut/plan` | 生成混剪方案 | 豆包 LLM + ffprobe |
| `/api/creative/sessions` | 创作记录存取 | 本地 JSON |
